## OpenClaw Integration Points

### Tools Used

| Tool | Purpose | Example |
|------|---------|---------|
| `browser` | Website scraping | `browser.navigate()`, `browser.screenshot()`, `browser.extractText()` |
| `llm` | Content analysis & generation | Analyze brand, generate HTML, write copy |
| `write` | Save files | Save HTML, JSON to output directory |
| `bash` | Deployment | `netlify deploy --prod` |

### Hooks

| Hook | Trigger | Purpose |
|------|---------|---------|
| `before_inference` | Every message | Detect company mentions, inject skill context |

### Gateway Methods

| Method | Purpose | Caller |
|--------|---------|--------|
| `signalToSite.process` | External trigger | Linkt webhook, API integrations |

## Error Handling

| Error | Handling |
|-------|----------|
| Site won't load | Use domain name only, skip detailed research |
| No about page | Continue without team info |
| LLM timeout | Retry once, then use template fallback |
| Netlify deploy fails | Save locally, report URL as local path |
| Rate limited | Queue and retry with backoff |

---

## Security Considerations

1. **No credentials stored** - API keys via environment only
2. **Approval gates** - Never auto-send without human review
3. **Public pages only** - Only scrapes publicly accessible content
4. **No login automation** - Never attempts to authenticate
5. **Rate limiting** - Respects reasonable request intervals

---

## Extending the System

### Adding a New Signal Type

1. Update pattern detection in `plugin/index.js`
2. Add signal-specific prompts in `prompts/`
3. Update Lobster workflow if needed

### Adding a New Channel

OpenClaw handles channel integration. The plugin automatically works with any connected channel.

### Adding a New Deployment Target

1. Add deployer logic to `registerTool(signal_to_site_deploy)`
2. Add config options in `openclaw.plugin.json`
3. Update `setup.sh` to prompt for credentials
